# Dead Dependencies & Dependency Tree Cleanup — Design Spec

**Date:** 2026-06-11
**Status:** Draft, awaiting user approval
**Target branch:** `chore/public-release-prep` (PR đang mở chưa merge)
**Scope:** 1 commit, surgical

## Goal

Tighten `package.json` for the public open-source release of `flux-ai` by
removing dependencies that have zero imports in the source tree, and by
moving MUI peer dependencies to the correct location in the manifest.

## Findings (audit on 2026-06-11)

Grep across all `*.ts` and `*.tsx` files under `app/`, `src/`, and
`e2e/`:

| Package           | Import count                                          | Verdict                                                      |
| ----------------- | ----------------------------------------------------- | ------------------------------------------------------------ |
| `axios`           | **0**                                                 | Dead. Next.js `fetch` is used everywhere.                    |
| `dayjs`           | **0**                                                 | Dead. `new Date().toISOString()` and native `Date` are used. |
| `@emotion/react`  | 0 (direct), but listed as peer dep of `@mui/material` | Required; should be `peerDependencies`                       |
| `@emotion/styled` | 0 (direct), but listed as peer dep of `@mui/material` | Required; should be `peerDependencies`                       |

All other dependencies have at least one direct import. Verified via
`git grep -lE "from ['\"](<lib>)['\"]"`.

## Stack coherence check (the "lộn xộn" concern)

A wider audit of the 21 libraries in `package.json` found **no stack
overlap**. Each library has a single, distinct role:

- **UI system:** `@mui/material`, `@mui/icons-material`,
  `@emotion/react`, `@emotion/styled` (one MUI ecosystem)
- **Animation:** `framer-motion` (7 files)
- **Server state:** `@tanstack/react-query` (4 files; `useChat`
  intentionally bypasses it because react-query doesn't fit streaming
  SSE responses)
- **Toasts:** `notistack` (1 file, provider in `app/providers.tsx`)
- **Charts:** `recharts` (1 file, `app/analytics/page.tsx`)
- **Markdown:** `react-markdown` + `rehype-highlight` + `remark-gfm`
  (3 packages, but this is the standard `react-markdown` plugin
  ecosystem — not duplication)
- **Date:** `dayjs` (dead) — would have been here if it were used
- **HTTP:** `axios` (dead) — `fetch` is used everywhere
- **Auth/DB/ORM:** `jose`, `bcryptjs`, `@prisma/*`, `pg` (server-side,
  single purpose)

There is **no library chaos** — only dead weight (axios, dayjs) and a
manifest metadata issue (emotion listed as `dependencies` instead of
`peerDependencies`).

## Decisions

| #   | Decision                                            | Rationale                                                                                                       |
| --- | --------------------------------------------------- | --------------------------------------------------------------------------------------------------------------- |
| 1   | Remove `axios`                                      | Zero imports, ~150 KB production bundle savings                                                                 |
| 2   | Remove `dayjs`                                      | Zero imports                                                                                                    |
| 3   | Move `@emotion/react` to `peerDependencies`         | MUI 5 peer dep; correct metadata                                                                                |
| 4   | Move `@emotion/styled` to `peerDependencies`        | MUI 5 peer dep; correct metadata                                                                                |
| 5   | Keep all other deps as-is                           | Each has a clear consumer and no overlap                                                                        |
| 6   | Keep `useChat` custom (not migrated to react-query) | Streaming SSE + AbortController is a deliberate state machine; react-query would add complexity without benefit |

## Out of Scope

- Refactoring `useChat` to use react-query (intentionally custom)
- Consolidating `react-markdown` ecosystem (standard plugin pattern)
- Replacing `notistack` (no duplication, works correctly)
- Upgrading MUI v5 → v6 (separate upgrade)
- Upgrading Next.js 15 → 16 (separate upgrade)
- Adding new dependencies

## Files Affected

- `package.json` — remove `axios`, `dayjs`; move `@emotion/react`,
  `@emotion/styled` from `dependencies` to `peerDependencies`
- `pnpm-lock.yaml` — regenerated by `pnpm install`

No source code changes. No test changes. No doc changes.

## Verification

After running `pnpm install`:

```bash
pnpm lint            # must exit 0
pnpm tsc --noEmit    # must exit 0
pnpm test            # 7/7 must pass
pnpm build           # must succeed
git grep -nE "from ['\"]axios['\"]" -- '*.ts' '*.tsx' || echo "axios gone"
git grep -nE "from ['\"]dayjs['\"]" -- '*.ts' '*.tsx' || echo "dayjs gone"
```

Plus a `pnpm build` to confirm `node_modules` resolves correctly with
the new peer-dep declaration.

## Risk Assessment

**Low.** All changes are metadata-only or removal of unused code:

- `axios` and `dayjs` are not imported anywhere; removing them cannot
  break the runtime.
- Moving `@emotion/*` to `peerDependencies` is a manifest-level change
  with no runtime impact. It may surface a peer-dep warning for
  downstream library consumers, but `flux-ai` is an app, not a library
  — the warning is informational.
- `pnpm install` will regenerate `pnpm-lock.yaml`; the new lockfile
  will be a few hundred lines smaller.

## Commit Plan

```

```

## Acceptance Criteria

1. `axios` and `dayjs` are gone from `package.json` and `pnpm-lock.yaml`
2. `@emotion/react` and `@emotion/styled` are listed in
   `peerDependencies` (not `dependencies`)
3. `pnpm install` succeeds without unresolved peer-dep errors
4. `pnpm lint`, `pnpm tsc --noEmit`, `pnpm test`, `pnpm build` all pass
5. `git grep` confirms zero source references to `axios` or `dayjs`
6. Single atomic commit on `chore/public-release-prep`
7. PR on GitHub auto-updates with the new commit
