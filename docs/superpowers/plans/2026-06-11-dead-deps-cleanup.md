## Task 1: Remove axios and dayjs; move @emotion/\* to peerDependencies

**Files:**

- Modify: `package.json` (lines 26-48, the `dependencies` block; add a new `peerDependencies` block)

- [ ] **Step 1: Read current `package.json` and verify the exact lines to edit**

Run: `cat package.json`

Expected: Confirms the 21-dependency layout (lines 26-48 are the `dependencies` block, 50-71 are `devDependencies`).

- [ ] **Step 2: Capture the lockfile size before changes (for bundle-savings claim)**

Run: `wc -l pnpm-lock.yaml`

Expected: A line count (e.g. 7000+). Record this number for the commit message body.

- [ ] **Step 3: Verify axios and dayjs have zero imports in source**

Run:

```bash
git grep -nE "from ['\"]axios['\"]|from ['\"]dayjs['\"]|require\(['\"]axios['\"]\)|require\(['\"]dayjs['\"]\)" -- '*.ts' '*.tsx' '*.js' '*.mjs' || echo "NO IMPORTS"
```

Expected: `NO IMPORTS`. If anything is found, STOP — re-evaluate.

- [ ] **Step 4: Remove axios and dayjs via pnpm**

Run:

```bash
pnpm remove axios dayjs
```

Expected: pnpm updates `package.json` and `pnpm-lock.yaml`. Two `dependencies` entries disappear. `pnpm-lock.yaml` shrinks.

- [ ] **Step 5: Verify the removals in `package.json`**

Run: `grep -E '"(axios|dayjs)"' package.json || echo "GONE"`

Expected: `GONE`. If either dep is still listed, re-run `pnpm remove <name>`.

- [ ] **Step 6: Move `@emotion/react` and `@emotion/styled` from `dependencies` to `peerDependencies`**

Open `package.json` in your editor. The two lines to remove from `dependencies` are:

```json
"@emotion/react": "^11.11.4",
"@emotion/styled": "^11.11.5",
```

After the existing `devDependencies` block (line 71), add a new `peerDependencies` block:

```json
"peerDependencies": {
  "@emotion/react": "^11.11.4",
  "@emotion/styled": "^11.11.5"
}
```

(Use the same version strings that were in `dependencies`.)

Use the Edit tool for this. **WARNING from Task 1 of the previous plan:** the Edit tool can trigger a PostToolUse formatter hook that re-prettifies surrounding lines. To avoid mixing this manifest-only change with a huge diff:

- Either use the Edit tool with very narrow `old_string` and `new_string` (just the two emotion lines + the peerDependencies addition).
- Or use `node -e` to do the read/modify/write in one shot.

Either approach is acceptable. The goal is to keep this commit's diff small (just `package.json` and `pnpm-lock.yaml`).

- [ ] **Step 7: Re-run `pnpm install` to sync the lockfile after the peer-dep move**

Run: `pnpm install`

Expected: pnpm may print a peer-dep warning (e.g. "@emotion/react listed in peerDependencies but not installed" if a downstream library tries to install `flux-ai` without emotion). This is informational, NOT an error. For the local install with `@mui/material` already a direct dep, pnpm will resolve emotion correctly because it remains in the lockfile.

If pnpm reports a hard error about missing peer deps, run `pnpm install --shamefully-hoist` as a fallback (rare).

- [ ] **Step 8: Verify `package.json` final shape**

Run:

```bash
node -e "
const p = require('./package.json');
console.log('dependencies:', Object.keys(p.dependencies).sort().join(', '));
console.log('peerDependencies:', Object.keys(p.peerDependencies).sort().join(', '));
console.log('devDependencies:', Object.keys(p.devDependencies).length, 'deps');
"
```

Expected output (approximate):

```
dependencies: @mui/icons-material, @mui/material, @prisma/adapter-pg, @prisma/client, @tanstack/react-query, bcryptjs, framer-motion, jose, next, notistack, pg, react, react-dom, react-markdown, recharts, rehype-highlight, remark-gfm
peerDependencies: @emotion/react, @emotion/styled
devDependencies: 21 deps
```

- [ ] **Step 9: Run the full verification suite**

Run:

```bash
pnpm lint
pnpm tsc --noEmit
pnpm test
pnpm build
```

Expected: All four exit with code 0. (Linting, type-checking, all 7 unit tests pass, and the production build succeeds.)

- [ ] **Step 10: Final grep — confirm no `axios` or `dayjs` references**

Run:

```bash
git grep -nE "(axios|dayjs)" -- ':!pnpm-lock.yaml' ':!package.json' || echo "NO REFS"
```

Expected: `NO REFS`. The only allowed references to `axios` or `dayjs` are in the lockfile's resolution history (which pnpm cleans up automatically) and in `package.json` itself (which should be gone — see Step 5).

- [ ] **Step 11: Stage and commit**

Run:

```bash
git add package.json pnpm-lock.yaml
git status
git diff --cached --stat
```

Expected: Only 2 files staged — `package.json` and `pnpm-lock.yaml`. No source code, no test files, no doc files.

- [ ] **Step 12: Create the commit**

Run:

```bash

Verified zero imports for axios and dayjs across app/, src/, and e2e/.

and @emotion/styled are MUI 5 peer dependencies — moving them to

version drift.

No source code changes. No runtime impact. ~150 KB production bundle
savings."
```

Expected: A new commit on `chore/public-release-prep` with exactly 2 files changed.
